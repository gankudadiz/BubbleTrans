## 目标

每次打开文件夹时自动跳转到上次阅读的页码，切换文件夹时实时更新阅读位置，让阅读心流不被打断。

## 设计思路

- 与 F6（最近打开文件夹）共享同一份持久化数据，复用 `config.json` 的读取/写入机制
- 存储结构：`config.json` 中新增 `last_positions` 字段，值为 `{ "文件夹路径": 页码索引 }`
- 仅记录最近 20 个文件夹的阅读位置，超出时淘汰最久未访问的记录
- 跳转逻辑：在 `load_file_list()` 完成后、渲染第一张图片前，检查是否有上次阅读位置

## 涉及文件

| 文件 | 改动类型 |
|------|----------|
| `src/utils/config.py` | 新增位置存取辅助函数 |
| `src/ui/window.py` | 切换文件夹时保存位置、启动时恢复位置 |

## 数据结构

```json
{
  "last_positions": {
    "F:/美漫/Supergirl/Volume 1": 15,
    "F:/美漫/Batwoman/Volume 3": 0,
    "F:/漫画/进击的巨人/Volume 34": 142
  }
}
```

- key：文件夹绝对路径（与 `current_folder` 完全一致）
- value：文件列表中的整数索引（0-based）
- 上限 20 条，超出时按写入时间淘汰最久未更新的条目

## 任务拆解

### 1.1 配置层扩展 — 位置读写

- 文件：`src/utils/config.py`
- 新增 `save_last_position(folder_path, page_index)` — 写入某个文件夹的阅读位置
- 新增 `get_last_position(folder_path)` — 读取某个文件夹的阅读位置，不存在返回 -1
- 内部逻辑：
  - `load_config()` 时若字段不存在则初始化为 `{}`
  - 写入时设置 `last_positions[folder_path] = page_index`
  - 超出 20 条时，按配置字典中的插入顺序淘汰最早条目（`popitem(last=False)`）
  - 通过 `save_config()` 持久化

### 1.2 保存阅读位置 — 切换文件夹/翻页时

- 文件：`src/ui/window.py`, `MainWindow.open_folder()`
- 在打开新文件夹前，先保存旧文件夹的当前位置：

```python
# open_folder() 顶部，folder 确认有效后
if self.current_folder and self.current_file_index >= 0:
    from utils.config import save_last_position
    save_last_position(self.current_folder, self.current_file_index)
```

- 同时保存旧位置后更新 `self.current_folder = folder`

### 1.3 恢复阅读位置 — 文件列表加载后

- 文件：`src/ui/window.py`, `MainWindow.load_file_list()`
- `load_file_list()` 末尾，文件列表加载完毕：

```python
from utils.config import get_last_position
saved_idx = get_last_position(self.current_folder)
if 0 <= saved_idx < self.file_list.count():
    self.file_list.setCurrentRow(saved_idx)
    self.current_file_index = saved_idx
    item = self.file_list.item(saved_idx)
    self.load_image(item)
else:
    # 位置无效（文件被删除/索引超限），加载第一页
    if self.file_list.count() > 0:
        self.file_list.setCurrentRow(0)
        item = self.file_list.item(0)
        self.load_image(item)
```

### 1.4 翻页时实时跟踪当前索引

- 文件：`src/ui/window.py`, `MainWindow.load_image()`
- 已有：`self.current_file_index = self.file_list.currentRow()` (L491)
- 无需额外改动，`current_file_index` 已在翻页/点击时同步更新

### 1.5 窗口关闭时保存最终位置

- 文件：`src/ui/window.py`, `MainWindow.closeEvent()`
- 在现有清理逻辑之前追加保存：

```python
if self.current_folder and self.current_file_index >= 0:
    from utils.config import save_last_position
    save_last_position(self.current_folder, self.current_file_index)
```

## 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 文件夹中图片被删除，上次索引超限 | `load_file_list()` 判断索引越界时回退到第一页 |
| 文件夹已不存在 | F6 打开时的路径校验会自动跳过，不影响 F7 |
| 首次打开某文件夹（无历史记录） | `get_last_position()` 返回 -1，加载第一页 |
| 用户清空了 `config.json` | 等效于首次打开，所有文件夹从第一页开始 |
| 快速切换多个文件夹 | 每次切换都保存旧位置，最终保存最新位置 |

## 验收标准

- [ ] 打开文件夹翻到第 N 页，关闭程序后重新打开该文件夹，自动跳到第 N 页
- [ ] 打开文件夹 A 翻到第 5 页，再打开文件夹 B 翻到第 10 页，关闭后重新打开 A 在第 5 页、B 在第 10 页
- [ ] `config.json` 中 `last_positions` 字段只保留最近 20 条，超出的被淘汰
- [ ] 文件夹图片被删除导致索引越界时，自动回退到第 1 页不报错
- [ ] `config.json` 缺失 `last_positions` 字段时，程序正常启动、按首次打开处理
