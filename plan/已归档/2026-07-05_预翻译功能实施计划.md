## 任务拆解

### P1. PrefetchManager 核心类

**构造与依赖**：
- `__init__(cache, max_concurrent=2)` — 注入 `TranslationCache` 实例，用于去重判断和结果写入

**核心方法**：
- `enqueue(paths: list[str])` → 去重后加入队列：跳过缓存命中（`cache.get()`）+ 已在队列中的 + 正在翻译的
- `promote(path: str)` → 从队列中移除指定页（用户 F5 翻译当前页时调用，该页从预翻译队列移除转由主翻译处理）；如果正在翻译中则不终止
- `resync(current_idx: int, total: int, prefetch_count: int)` → 重新计算目标范围 [current+1, min(current+N, total-1)]，**移除队列中不在范围内的项**，追加范围内新增的项，已存在的保留
- `_process_queue()` → 取最多 `max_concurrent` 个页面启动 `TranslationWorker`
- `clear()` → 清空队列 + 终止所有运行中 worker

**完成回调**（全部在 Qt 信号槽机制下于主线程执行，天然线程安全）：
- Worker 完成 → `cache.set()` 写入翻译缓存 → 从 `active` 中移除 → 发射 `page_completed(image_path)` → 从队列取下一个 → 发射 `progress_changed(completed, total)`
- 队列 + active 均为空 → 发射 `all_completed()`
- Worker 报错 → 静默跳过该页，继续处理后续

**信号设计**：
- `progress_changed(int completed, int total)` — 进度更新，供状态栏显示
- `page_completed(str image_path)` — 单页完成，供 window 判断是否需要刷新 UI（如当前正好显示该页）
- `all_completed()` — 全部完成，供状态栏显示完成提示

### P2. UI 改造（window.py）

**按钮与菜单**：
- 右侧面板「翻译当前页」右侧新增 `预翻译 ▼` 下拉按钮（QToolButton + QMenu）
- 菜单选项：下 1 页 / 下 3 页 / 下 5 页
- 点击后调用 `prefetch_manager.enqueue(paths)`，path 列表由当前页索引 + 用户选择 N 计算出
- 最后一页时按钮置灰；共 1 页时不显示

**快捷键**：
- `Ctrl+Shift+F5` → 等同于「预翻译下 3 页」
- `keyPressEvent` 中新增分支

**PrefetchManager 集成**：
- `__init__` 中实例化 `PrefetchManager(self.translation_cache, max_concurrent=load_config("prefetch_concurrent", 2))`
- 连接信号：
  - `progress_changed` → 更新状态栏：`📦 预翻译: {completed}/{total} 页已完成`
  - `all_completed` → 状态栏提示 `📦 预翻译完成: {total} 页已缓存`，3 秒后消失（QTimer.singleShot）
  - `page_completed(path)` → 判断：如果 `path == self.canvas._current_image_path`，从缓存读取该页结果，刷新右侧面板（调用 `on_translation_finished` 但标记 `from_prefetch=True` 以区别缓存回写逻辑）

**F5 主翻译与预翻译协同**：
- F5 按下时 → `prefetch_manager.promote(current_image_path)` 先从预翻译队列移出
- 然后走原有的 `translate_current_page()` 流程（不变）

**_set_translating 多 worker 适配**：
- 当前是二态（翻译中 / 空闲），引入预翻译后需要区分"主翻译状态"和"预翻译状态"
- 方案：`_set_translating` 语义不变，仅控制**主翻译**的 UI 状态（按钮禁用、骨架屏）
- 预翻译的进度通过 `PrefetchManager` 的 `progress_changed` 信号独立更新状态栏，不影响按钮/骨架屏
- 主翻译和预翻译同时进行时：按钮显示"翻译中…" + 骨架屏（主翻译），状态栏显示预翻译进度

**中断与清理**：
- `closeEvent` → `prefetch_manager.clear()`
- `_load_source`（切换文件夹/压缩包）开头 → `prefetch_manager.clear()`

### P3. Settings 配置项

- `prefetch_enabled`：复选框「翻页时自动预翻译」（默认 false）
- `prefetch_count`：下拉框「预翻译页数」选项 1 / 3 / 5（默认 3）
- `prefetch_concurrent`：下拉框「并发数」选项 1 / 2 / 3（默认 2）
- 所有字段带默认值，写入 `config.json`

### P4. 自动预翻译逻辑

**触发时机**：`load_image` 末尾（图片异步加载完成的 `_on_image_loaded` 回调中），确保当前页已渲染

**完整 resync 流程**：
1. 读取 `prefetch_enabled`，如关闭则跳过
2. 调用 `prefetch_manager.resync(current_idx, total, prefetch_count)`，由 PrefetchManager 内部计算目标范围并同步队列：
   - 目标范围 = [current_idx+1, min(current_idx+prefetch_count, total-1)]
   - 遍历当前队列，移除 index < target_start 或 index > target_end 的项
   - 对范围内未在队列/active/缓存中的页面，追加到队列

**注意**：`resync` 只操作队列中未开始的项，`active` 中正在翻译的项不中断。

## 实施顺序

1. P1 PrefetchManager 核心类（无依赖，可独立开发）
2. P3 Settings 配置项（无依赖，可独立开发）
3. P2 UI 改造 + P4 自动预翻译（依赖 P1、P3）

## 验收标准

- 手动预翻译 N 页：状态栏显示正确进度，完成后全部写入缓存
- 自动预翻译：翻页后自动触发，快速连翻时队列正确同步（旧项移除 + 新项追加）
- 缓存命中：预翻译后翻到该页，译文和总结瞬间显示
- 尾部边界：最后一页按钮正确置灰，预翻译范围不越界
- 预翻译进行中按 F5：当前页从预翻译队列移除，主翻译正常完成，预翻译继续处理剩余页
- 中断清理：切换文件夹后队列清空、worker 终止
- 主翻译不受影响：F5 翻译当前页按钮行为、骨架屏、UI 面板更新与现有逻辑完全一致
- 状态栏完成提示：预翻译全部完成后显示 `📦 预翻译完成: N 页已缓存`，3 秒后消失
