## 目标

向用户展示可用的快捷键，降低上手门槛。通过首次引导浮层 + 状态栏常驻提示 + Settings tooltip 三层覆盖。

| 文件 | 改动类型 |
|------|----------|
| `src/ui/window.py` | 补 F5 翻译当前页、快捷键引导浮层、状态栏提示、Settings tooltip |
| `src/ui/canvas.py` | 补 Esc 取消框选 |

- 右键拖拽：框选翻译区域（`canvas.py:443`）

## 任务拆解

### 4.0 前置：补齐缺失的快捷键（先于 4.1 执行）

> **原因**：引导浮层中列出的快捷键必须先能正常使用，否则是虚假宣传。

| 快捷键 | 文件 | 改动 |
|--------|------|------|
| `Esc` 取消框选 | `src/ui/canvas.py` | `keyPressEvent` 中新增 `Qt.Key.Key_Escape`：仅在 `self.is_selecting` 为 true 时，隐藏 rubber_band 并重置 `is_selecting = False` |
| `F5` 翻译当前页 | `src/ui/window.py` | `keyPressEvent` 中新增 `Qt.Key.Key_F5`：调用 `self.translate_current_page()` |

### 4.1 首次启动快捷键引导浮层

- **实现方式**：`ShortcutOverlay(QDialog)` + `FramelessWindowHint` + `WA_TranslucentBackground`
  - 不采用 QWidget + QGraphicsOpacityEffect（Windows 下曾有边框消失/文字模糊的已知问题）
  - `exec()` 自带模态事件循环，关闭前主窗口不响应，对首次引导是正面行为
- **触发时机**：不在 `__init__` 中直接 `exec()`，而是在 `__init__` 末尾用 `QTimer.singleShot(500, self._maybe_show_shortcut_overlay)` 延迟触发，确保主窗口已完整渲染
- **触发条件**：`config.json` 中 `shortcut_hint_shown` 字段不存在或为 `false`
- **关闭行为**：勾选"不再提示"后关闭 → 调用 `save_config({"shortcut_hint_shown": True})`
  - `save_config` 是 merge 写入，不会覆盖 config.json 中的其他字段
- **样式**：半透明遮罩 + 居中暗色卡片，与主窗口风格一致

快捷键列表（本次补齐后全部可用的快捷键）：
| 快捷键 | 功能 | 状态 |
|--------|------|------|
| `←` / `→` | 上一页 / 下一页 | 已有 |
| 右键拖拽 | 框选翻译区域 | 已有 |
| 滚轮 | 缩放图片 | 已有 |
| `Esc` | 取消框选 | 4.0 新增 |
| `F5` | 翻译当前页 | 4.0 新增 |

### 4.2 状态栏常驻快捷键提示

- 状态栏右侧（或固定位置）添加 label：`← → 翻页 | 右键框选 | 滚轮缩放`
- 使用 `QStatusBar.addPermanentWidget()` 保证位置固定
- 字体略小于状态栏主文本，颜色偏灰

### 4.3 Settings 按钮 tooltip

- Settings 按钮添加 `setToolTip("设置 (快捷键、API 配置)")`
- hover 时显示简要说明

## 验收标准

- [ ] Press `Esc` 在框选进行中时能取消当前框选，且只有在框选时生效
- [ ] Press `F5` 能触发翻译当前页（与点击"翻译当前页"按钮行为一致）
- [ ] 首次启动（或 `shortcut_hint_shown` 为 false）时弹出快捷键引导浮层
- [ ] 勾选"不再提示"并关闭后，下次启动不再弹出
- [ ] 状态栏始终显示核心快捷键提示（`← → | 右键框选 | 滚轮缩放`）
- [ ] Settings 按钮 hover 时显示 tooltip
- [ ] 引导浮层样式美观、与主窗口风格一致
- [ ] 引导浮层中列出的快捷键全部可用且描述正确
