## 目标

记住最近打开的文件夹/压缩包路径，通过下拉菜单快速回到上次阅读的漫画，并支持启动时自动恢复。

## 涉及文件

| 文件 | 改动类型 |
|------|----------|
| `src/ui/window.py` | Open Folder 按钮下拉菜单 + 自动恢复逻辑 |
| `src/utils/config.py` | 新增 `recent_folders` 存储支持 |
| `config.json` | 新增 `recent_folders` 和 `auto_open_last` 字段 |

## 方案设计

### 存储
- `config.json` 新增字段：
  ```json
  {
    "recent_folders": ["D:/manga/vol1", "D:/manga/vol2.cbz"],
    "auto_open_last": true
  }
  ```
- 最多保留 10 条，新条目插入到最前，重复路径移到最前（去重）

### UI
- Open Folder 按钮改为带下拉箭头的 `QToolButton`（`setPopupMode(QToolButton.MenuButtonPopup)`）
- 下拉菜单列出最近路径
  - 每项显示路径（过长中间截断）
  - 底部"清除历史"选项
  - 菜单项右侧可 hover 显示完整路径 tooltip

### 自动恢复
- 程序启动时检查 `auto_open_last`：
  - 若为 `true` 且 `recent_folders` 非空
  - 取第一条，检查路径是否存在
  - 存在则自动打开；不存在则从列表移除并继续检查下一条

## 任务拆解

### 6.1 config 扩展
- `src/utils/config.py`：确保 `load_config()` 的默认值中包含 `recent_folders: []` 和 `auto_open_last: true`
- 新增工具方法 `add_recent_folder(path)`：
  - 从 `recent_folders` 中移除已有相同路径
  - 插入到列表首位
  - 截断到 10 条
  - `save_config()`

### 6.2 Open Folder 按钮改造
- `src/ui/window.py`：
  - 将 Open Folder 按钮替换为 `QToolButton` + `QMenu`
  - 按钮主体点击 → 弹出文件夹选择对话框（原有逻辑）
  - 下拉箭头点击 → 展开最近路径菜单
  - 菜单项点击 → 直接打开该路径
  - 底部 `addSeparator()` + "清除历史" action

### 6.3 打开路径时记录
- `open_source()` / `open_folder()` 成功打开后调用 `add_recent_folder(path)`
- 无论是文件夹、`.cbz`、`.zip`、`.cbr` 都记录

### 6.4 自动恢复
- `MainWindow.__init__()` 末尾（UI 就绪后）
  - 若 `auto_open_last` 且 `recent_folders` 非空
  - 遍历列表找到第一个存在的路径，自动打开
  - 不存在的路径从列表中移除并 `save_config()`

### 6.5 设置入口（可选）
- Settings 对话框增加 `auto_open_last` 复选框
- 若改动成本低则一并做，否则后续版本

## 验收标准

- [ ] 打开一个文件夹后，该路径出现在下拉菜单中
- [ ] 打开多个文件夹后，最近打开的排在最前，最多 10 条
- [ ] 相同路径重复打开 → 移到最前，不重复
- [ ] 下拉菜单点击路径 → 直接打开该文件夹
- [ ] "清除历史" → 下拉菜单清空
- [ ] 启用自动恢复时，启动后自动打开上次文件夹
- [ ] 自动恢复时路径不存在 → 自动移除并跳过
- [ ] `.cbz` / `.zip` / `.cbr` 路径同样记录和恢复
