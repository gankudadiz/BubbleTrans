## 目标

支持直接打开 `.cbz` / `.zip` / `.cbr` 压缩包，程序内部解压到临时目录后按现有翻页流程使用，关闭时自动清理。

## 涉及文件

| 文件 | 改动类型 |
|------|----------|
| `src/utils/archive.py`（新建） | 压缩包解压与文件列表提取 |
| `src/ui/window.py` | 打开文件对话框增加压缩包过滤；关闭时清理临时目录 |

## 方案设计

- `.cbz` = zip 格式（Python `zipfile` 标准库）
- `.zip` = 同上
- `.cbr` = rar 格式，优先尝试 `rarfile` 库（纯 Python），不可用时提示安装
- 解压到 `tempfile.mkdtemp(prefix="bubbletrans_")` 临时目录
- 解压后过滤常见图片后缀（`.jpg/.jpeg/.png/.webp/.bmp/.gif`）
- 按文件名自然排序作为文件列表
- `MainWindow` 退出时清理临时目录

## 任务拆解

### 2.1 创建 `ArchiveHandler` 类
- 文件：`src/utils/archive.py`
- `__init__(archive_path)`
  - 根据扩展名选择解压器：`.cbz`/`.zip` → `zipfile`，`.cbr` → `rarfile`
  - 创建临时目录
- `extract()` — 解压全部图片文件到临时目录，返回图片路径列表
- `close()` — 删除临时目录（`shutil.rmtree`）
- `get_image_list()` — 不解压，仅列出压缩包内的图片文件名
- 错误处理：损坏的压缩包、不支持的格式、`rarfile` 未安装

### 2.2 接入打开流程
- `src/ui/window.py`：
  - `open_folder()` 改为 `open_source()`，支持文件夹和压缩包
  - 文件对话框增加过滤器：`漫画压缩包 (*.cbz *.cbr *.zip);;所有文件 (*)`
  - 打开压缩包时调用 `ArchiveHandler.extract()`
  - 将临时目录路径作为"当前文件夹"传给图片加载逻辑
  - 文件列表显示压缩包内文件名（不暴露临时路径）

### 2.3 生命周期管理
- `MainWindow` 持有 `self._archive_handler: Optional[ArchiveHandler]`
- `MainWindow.closeEvent()` 中调用 `archive_handler.close()` 清理临时目录
- 打开新文件前，先关闭旧压缩包

### 2.4 依赖处理
- `zipfile`：Python 标准库，无需额外安装
- `rarfile`：`pip install rarfile`，添加到 `requirements.txt`
- `.cbr` 无 `rarfile` 时给出提示并回退到仅支持 `.cbz`/`.zip`

## 验收标准

- [ ] 打开 `.zip` 文件同上

- [ ] 关闭程序后临时目录被清理
- [ ] 打开新压缩包时旧临时目录被清理
