## 目标

每次重启程序后，已翻译过的页面直接从本地缓存恢复，避免重复调用 API。缓存命中后秒级恢复原文/译文/总结/备注。

## 涉及文件

| 文件 | 改动类型 |
|------|----------|
| `src/utils/cache.py`（新建） | 缓存管理器 `TranslationCache` |
| `src/engine/llm.py` | `translate_image()` 调用前后接入缓存 |
| `src/ui/window.py` | `on_translation_finished()` 写缓存；初始化时加载缓存 |

## 缓存设计

- **键**：`hashlib.md5(图片绝对路径 + 文件修改时间戳).hexdigest()`
- **值**：`{ "original": str, "translated": str, "summary": dict, "timestamp": float }`
- **存储**：`config.json` 同级目录 `translation_cache.json`
- **容量**：上限 500 条，LRU 淘汰（`OrderedDict` 天然支持）
- **加载**：程序启动时一次性加载到内存，后续只写磁盘不读磁盘

## 任务拆解

### 1.1 创建 `TranslationCache` 类
- 文件：`src/utils/cache.py`
- `__init__(cache_path)` — 指定缓存文件路径
- `load()` — 从 JSON 加载到 `OrderedDict`
- `get(image_path, mtime)` — 生成 key 查缓存，命中则移到末尾
- `set(image_path, mtime, data)` — 写入缓存，超出 500 条则 `popitem(last=False)`
- `save()` — 持久化到 JSON
- 处理文件不存在、JSON 损坏的异常

### 1.2 接入翻译流程 — 查缓存
- `src/engine/llm.py`：`LLMEngine.__init__()` 传入并持有 `TranslationCache` 实例
- `translate_image(image_path)` 方法开头：
  - 读取文件 mtime
  - 调用 `cache.get(image_path, mtime)`
  - 命中则直接返回缓存数据，跳过 API 调用
  - 返回结果中增加标志位（区分缓存命中/API 新译）

### 1.3 接入翻译流程 — 写缓存
- `src/ui/window.py`：`on_translation_finished()` 中
  - 翻译成功后调用 `cache.set(image_path, mtime, data)`
  - 调用 `cache.save()` 持久化
- 注意：仅 API 新译结果写缓存，缓存命中无需重复写

### 1.4 初始化与生命周期
- `MainWindow.__init__()` 中：
  - 根据 `config.json` 路径推导 `translation_cache.json` 路径
  - 创建 `TranslationCache` 实例
  - 传入 `LLMEngine`
- 缓存只在翻译成功时自动 `save()`，无需定时保存

## 验收标准

- [x] 翻译一张图片后重启程序，再次打开该图，原文/译文/总结立即显示（无需等待 API）
- [x] 修改图片后（mtime 变化），翻译时走 API 而非旧缓存
- [x] 缓存超 500 条后，最久未访问的条目被淘汰
- [x] `translation_cache.json` 损坏或不存在时，程序正常启动不报错
- [x] 缓存文件位于 `config.json` 同级目录
