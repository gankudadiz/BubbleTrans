## 目标

翻译过程中给用户清晰的分阶段进度反馈，消除"不知道还要等多久"的焦虑感，同时通过骨架屏减少右侧面板的"白屏等待"。

## 涉及文件

| 文件 | 改动类型 |
|------|----------|
| `src/ui/window.py` | 状态栏分阶段文本 + 翻译按钮动画 + 骨架屏 |
| `src/engine/llm.py` | `translate_image()` 增加阶段回调/信号 |

## 方案设计

当前流程（`TranslationWorker.run()` → `llm_engine.translate_image()`）是单次调用、单次返回。需要拆分为阶段信号，让 UI 层能感知当前进度：

```
编码图片 → 等待 API 响应 → 解析结果 → 生成总结
```

### 3.1 翻译流程阶段信号化

在 `LLMEngine.translate_image()` 中增加回调参数或改为 yield：

- 方案 A（轻量）：在 `translate_image()` 内部各关键步骤前后通过 `QObject` 信号 emit 阶段名
  - `signal.stage_changed.emit("encoding")` — 编码图片前
  - `signal.stage_changed.emit("waiting")` — API 调用中
  - `signal.stage_changed.emit("parsing")` — 解析结果中
  - `signal.stage_changed.emit("summary")` — 生成总结中（实际上总结是一起返回的，这里可以处理后续步骤）

- 方案 B（更简）：在 `TranslationWorker.run()` 中分步发送信号，将 `translate_image` 调用前后的阶段暴露出来。但 API 调用内部无法拆分，所以只能在调用前后发信号。

建议方案 B（改动最小）：
```
TranslationWorker.run():
  emit stage="正在编码图片…"
  encode_image()  # 如果暴露出来
  emit stage="等待 API 响应…"
  result = llm_engine.translate_image()
  emit stage="正在解析结果…"
  emit stage="完成"
```

如果 `encode_image` 不单独暴露，则简化为：
```
emit stage="正在翻译…" (编码+API)
result = translate_image()
emit stage="正在解析…"
emit stage="完成"
```

### 3.2 状态栏分阶段提示

- 在 `MainWindow` 中添加 `QStatusBar` label 更新方法
- `TranslationWorker` 发射 `stage_changed(str)` 信号
- `on_translation_stage_changed()` → 更新 statusbar 文本

### 3.3 翻译按钮加载动画

- 翻译按钮使用 `QPushButton` 或 `QToolButton`
- 翻译中设置按钮 `setEnabled(False)` + 替换图标为旋转动画
- 使用 `QPropertyAnimation` + 旋转的 `QIcon` 或 `QMovie`（GIF）
- 翻译完成后恢复原图标

### 3.4 右侧面板骨架屏

- 原文/译文区、总结区在等待翻译时显示灰色占位块
- 实现：在原文区、译文区、总结区各放置一个半透明灰色矩形 `QFrame`（或 `QLabel` 带灰色背景）
- 翻译完成后隐藏骨架屏，显示实际内容
- 骨架屏样式参考：`background: #e0e0e0; border-radius: 4px;` 带轻微脉冲动画（可选）

## 任务拆解

### 3.1 阶段信号
- `TranslationWorker` 新增 `stage_changed = pyqtSignal(str)`
- `run()` 方法中在关键步骤前后 emit 阶段文本
- 阶段文本使用中文：`"正在编码图片…"` `"等待 API 响应…"` `"正在解析结果…"` `"完成"`

### 3.2 状态栏更新
- `MainWindow.__init__()` 中连接 `worker.stage_changed` → 更新 statusbar
- 确保状态栏 label 在翻译完成后恢复默认文本

### 3.3 按钮动画
- 翻译按钮在翻译中替换为旋转图标
- 简单方案：使用 Unicode 旋转字符或预制的 GIF 作为 `QMovie`
- 翻译完成 → 恢复原图标 + enabled

### 3.4 骨架屏
- 在原文区、译文区、总结区的 `QTextEdit`/`QLabel` 上层覆盖骨架 `QFrame`
- 骨架屏 `show()` / `hide()` 随翻译状态切换
- 当前已有内容时无需显示骨架屏（仅首次翻译或清空后翻译时显示）

## 验收标准

- [ ] 翻译进行时，状态栏依次显示"正在编码图片…" → "等待 API 响应…" → "正在解析结果…" → 翻译完成
- [ ] 翻译按钮在翻译中不可点击，显示加载动画
- [ ] 翻译完成后按钮恢复可点击，动画消失
- [ ] 右侧面板翻译等待期间显示骨架屏占位
- [ ] 翻译完成后骨架屏消失，内容正常显示
- [ ] 翻译失败时各状态正常恢复
