## 目标

在文件列表顶部增加搜索框，支持实时模糊过滤文件名，解决几百张图片时滚动找图的低效问题。

## 涉及文件

| 文件 | 改动类型 |
|------|----------|
| `src/ui/window.py` | 文件列表面板增加搜索框 + 过滤逻辑 |

## 方案设计

当前文件列表使用 `QListWidget`（需确认实际 widget 类型），搜索功能实现：

- 在文件列表上方插入 `QLineEdit` 搜索框
- `textChanged` 信号触发实时过滤
- 过滤逻辑：遍历所有 item，`file_name.lower().contains(query.lower())` 的显示，不匹配的隐藏
- 匹配文本高亮：考虑使用 `QStyledItemDelegate` 在 item 文字中高亮匹配子串
- 清空搜索框 → 恢复完整列表

### 搜索框 UI 布局

```
┌──────────────────────┐
│ 🔍 搜索文件名…       │  ← QLineEdit with placeholder
├──────────────────────┤
│ page_001.jpg         │
│ page_002.jpg         │
│ …                   │
└──────────────────────┘
```

- `QLineEdit` 左侧加搜索图标（Unicode 🔍 或 QIcon）
- placeholder 文本："搜索文件名…"
- 搜索框 `clearButtonEnabled = True`（右侧 X 按钮清空）
- 快捷键 `Ctrl+F` 聚焦搜索框

## 任务拆解

### 5.1 确认文件列表实现
- 读取 `src/ui/window.py`，确认文件列表使用的 widget 类型（`QListWidget` / `QListView` / 自定义）
- 确认文件列表的填充逻辑所在方法
- 确认文件列表的点击/选择事件绑定

### 5.2 添加搜索框
- 在文件列表面板顶部插入 `QLineEdit`
  - `setPlaceholderText("搜索文件名…")`
  - `setClearButtonEnabled(True)`
  - 连接 `textChanged` 信号
- 搜索框与文件列表之间用 `QSplitter` 或 `QVBoxLayout` 组织

### 5.3 实现过滤逻辑
- `filter_file_list(query: str)` 方法
  - 遍历 `QListWidget` 所有行
  - `item.setHidden(query not in item.text().lower())`
  - 空 query → 全部显示
- 注意性能：百张级别遍历无压力，千张级别同样可接受

### 5.4 高亮匹配文本（可选增强）
- 使用 `QStyledItemDelegate` 覆写 `paint()` 方法
- 在 item 文本中用 `QTextCharFormat` 高亮匹配子串
- 若实现成本高可先跳过，仅做过滤

### 5.5 快捷键绑定
- `Ctrl+F` → 聚焦搜索框
- `Esc` → 清空搜索框并失去焦点
