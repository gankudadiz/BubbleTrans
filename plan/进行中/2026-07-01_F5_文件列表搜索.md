## 目标

在文件列表顶部增加搜索框，支持实时模糊过滤文件名，解决几百张图片时滚动找图的低效问题。

## 涉及文件

| 文件 | 改动类型 |
|------|----------|
| `src/ui/window.py` | 文件列表面板增加搜索框 + 过滤逻辑 |

## 方案设计

当前文件列表使用 `QListWidget`（window.py:436），搜索功能实现：

- 在文件列表上方插入 `QLineEdit` 搜索框
- `textChanged` 信号触发实时过滤
- 过滤逻辑：遍历所有 item，`query in item.text().lower()` 的显示，不匹配的 `setHidden(True)`
- 匹配文本高亮：考虑使用 `QStyledItemDelegate` 在 item 文字中高亮匹配子串
- 清空搜索框 → 恢复完整列表
- 翻页时保留过滤状态，自动跳过隐藏项（见 5.3）

### 搜索框 UI 布局

```
┌──────────────────────┐
│ 🔍 搜索文件名…       │  ← QLineEdit with placeholder
├──────────────────────┤
│ page_001.jpg         │

└──────────────────────┘
```

- `QLineEdit` 左侧加搜索图标（Unicode 🔍 或 QIcon）
- placeholder 文本："搜索文件名…"
- 搜索框 `clearButtonEnabled = True`（右侧 X 按钮清空）
- 快捷键 `Ctrl+F` 聚焦搜索框

## 任务拆解

### 5.1 确认搜索框嵌入位置
- 读取 `load_file_list`（window.py:649）和文件列表初始化（window.py:436-439）
- 确定容器 widget 的插入点（将 `splitter.addWidget(self.file_list)` 改为加入容器）
- 确认文件列表的点击/选择事件绑定不受重构影响

### 5.2 添加搜索框
- **关键：左侧面板需要重构为容器 widget**
  - 新建容器 `QWidget` 作为左侧面板
  - 容器内部用 `QVBoxLayout` 包裹搜索框 + `QListWidget`
  - 将容器（而非 QListWidget 本身）加入 `QSplitter`
- 在容器顶部插入 `QLineEdit`
  - `setPlaceholderText("搜索文件名…")`
  - `setClearButtonEnabled(True)`
  - 连接 `textChanged` 信号

### 5.3 实现过滤逻辑
- `filter_file_list(query: str)` 方法
  - 遍历 `QListWidget` 所有行
  - `item.setHidden(query not in item.text().lower())`
  - 空 query → 全部显示
- 注意性能：百张级别遍历无压力，千张级别同样可接受
- **翻页时跳过隐藏项**：修改 `_navigate_page`（window.py:926），翻页循环中检查 `item.isHidden()`，跳到下一个可见项；边界外则不翻页

### 5.4 高亮匹配文本（可选增强）
- 使用 `QStyledItemDelegate` 覆写 `paint()` 方法
- 在 item 文本中用 `QTextCharFormat` 高亮匹配子串
- 若实现成本高可先跳过，仅做过滤

### 5.5 快捷键绑定
- `Ctrl+F` → 用 `QShortcut(QKeySequence("Ctrl+F"), self, self._focus_search_box)` 绑定聚焦（避免污染 `keyPressEvent`）
- `Esc` → 清空搜索框并失去焦点，**上下文绑定到搜索框本身**：
  ```python
  self.search_box = QLineEdit(...)
  QShortcut(QKeySequence("Escape"), self.search_box, self._clear_search)
  ```
  - 原理：`QShortcut` 的 context widget 设为 `self.search_box`，只有搜索框（或其子控件）拥有焦点时 Esc 才触发
  - Qt 内部在 shortcut 层拦截 Esc 按键事件，不经过 `keyPressEvent`，避免与画布/主窗口的 Esc 冲突
