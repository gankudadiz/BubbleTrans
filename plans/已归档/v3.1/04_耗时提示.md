## 1. 目标

批量翻译范围 > 15 页时，在启动翻译前弹出耗时预估提示，让用户做出知情选择（继续或缩减范围）。≤ 15 页不提示，直接开始。

## 2. 设计方案

### 2.1 触发条件与公式

```
if total_pages > 15:
    弹提示
else:
    直接开始翻译
```

**预估耗时**：

```
estimated_seconds = total_pages * avg_time_per_page / concurrency
estimated_minutes_min = floor(estimated_seconds / 60 * 0.7)   # 偏乐观
estimated_minutes_max = ceil(estimated_seconds / 60 * 1.3)    # 偏保守

显示: "将翻译 XX 页，预计约 X–XX 分钟，请耐心等待"
```

**参数取值**：

| 参数 | 值 | 说明 |
|------|-----|------|
| `avg_time_per_page` | 8 秒 | 保守估计（含 API 往返 + 解析），可通过配置调整 |
| `concurrency` | 2（默认） | 复用 settings 中预翻译并发数 |
| `threshold` | 15 页 | 低于此值不提示 |
| `multiplier_low` | 0.7 | 乐观系数 |
| `multiplier_high` | 1.3 | 悲观系数 |

**示例**：
- 21 页，并发 2 → 21 × 8 / 2 = 84 秒 → 显示"预计约 1–2 分钟"
- 50 页，并发 2 → 50 × 8 / 2 = 200 秒 → 显示"预计约 2–4 分钟"

### 2.3 插入位置

在 `window._show_batch_dialog()` 中，范围选择对话框关闭后、启动批量翻译前：

```python
def _show_batch_dialog(self):
    # ... 弹出 BatchRangeDialog，获取 start, end ...
    
    page_count = end - start + 1
    
    # 耗时提示（> 15 页时）
    if page_count > BATCH_TIME_THRESHOLD:
        if not self._confirm_batch_start(page_count):
            return  # 用户取消
    
    # 启动批量翻译
    self.batch_manager.start(...)
```

### 2.4 辅助方法

```python
BATCH_TIME_THRESHOLD = 15       # 超过此页数弹出耗时提示
PAGE_AVG_TIME_SEC = 8           # 单页平均耗时（秒）

def _confirm_batch_start(self, page_count: int) -> bool:
    """弹出批量翻译耗时确认对话框"""
    concurrency = self.settings.get("prefetch_concurrency", 2)
    estimated_secs = page_count * PAGE_AVG_TIME_SEC / concurrency
    
    min_min = max(1, int(estimated_secs / 60 * 0.7))
    max_min = max(1, int(estimated_secs / 60 * 1.3) + 1)
    
    if max_min <= 1:
        time_str = "不到 1 分钟"
    else:
        time_str = f"{min_min}–{max_min} 分钟"
    
    msg = (f"将翻译 {page_count} 页\n\n"
           f"预计约 {time_str}，请耐心等待\n\n"
           f"提示：可先翻译少量页面，确认翻译质量满意后再继续。")
    
    reply = QMessageBox.question(
        self, "批量翻译确认", msg,
        QMessageBox.StandardButton.Yes | QMessageBox.StandardButton.No,
        QMessageBox.StandardButton.Yes
    )
    return reply == QMessageBox.StandardButton.Yes
```

## 5. 风险点

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 预估时间与实际偏差大 | 中 | 用户预期不符 | 使用区间而非单点预估；公式偏保守 |
| 并发数为 0 导致除零 | 极低 | 程序崩溃 | `concurrency` 取值时 `max(1, ...)` 兜底 |
