## 1. 目标

翻译第 N 页时，将第 N-5 到 N-1 页的剧情总结（`plot` 字段）作为"前情提要"注入 LLM prompt，解决跨页翻译时的角色名漂移和剧情断层问题。

## 2. 当前状态

### 2.1 当前翻译链路

```
window.F5 → controller.translate_page(path)
              → _start_worker(path)
                → TranslationWorker(image_path)
                  → llm_engine.translate_image(image_path)
                    → 构建 messages（system + user[image]）
                    → API 调用
```

**每一页的翻译完全独立**，LLM 不知道前文讲什么。

### 2.2 涉及的模块

| 文件 | 当前行数 | 改动范围 |
|------|---------|---------|
| `engine/llm.py` | 665 行 | `translate_image()` 签名 + `_build_system_prompt()` 调用处 |
| `engine/workers.py` | 55 行 | `TranslationWorker.__init__` + `run()` |
| `engine/translation_controller.py` | 165 行 | `translate_page()` / `translate_region()` 增加 context 参数 |
| `utils/database.py` | PDR-01 产物 | 已预留 `get_page_range()` |

### 2.3 上下文数据来源

v3.1 PDR 设计决策：使用 `plot`（剧情总结）而非全文翻译做上下文。

- 数据已在缓存中：每次翻译完成后，`summary.plot` 随缓存写入
- `plot` 信息密度高：AI 主动总结，每页 60-100 字
- 噪音低：全文翻译包含拟声词、格式标记等

## 3. 设计方案

### 3.1 新建：`src/engine/context.py`

```python
class ContextEngine:
    """滑动窗口上下文构建器
    
    从 SQLite 中查询前 N 页的 plot 字段，拼接为"前情提要"文本。
    """
    
    def __init__(self, database):
        """
        参数:
            database: utils.database.Database 实例
        """
        self._db = database
    
    def build_context(self, folder_path: str, current_page: int,
                      window_size: int = 5, max_chars: int = 800) -> str | None:
        """
        为当前页构建上下文文本
        
        参数:
            folder_path: 图片所在文件夹路径
            current_page: 当前页码（0-based）
            window_size: 滑动窗口大小（取前多少页）
            max_chars: 上下文最大字符数
        
        返回:
            上下文文本，无可用上下文时返回 None
        """
        if current_page <= 0:
            return None
        
        start = max(0, current_page - window_size)
        end = current_page - 1
        
        rows = self._db.get_page_range(folder_path, start, end)
        
        if not rows:
            return None
        
        # 构建上下文
        lines = ["【前情提要】"]
        total_chars = 0
        
        for row in rows:
            plot = (row.get("plot") or "").strip()
            if not plot:
                continue
            page_label = row.get("page_index", "?")
            line = f"第{page_label}页：{plot}"
            if total_chars + len(line) > max_chars:
                lines.append("（更早的剧情已省略）")
                break
            lines.append(line)
            total_chars += len(line)
        
        if len(lines) == 1:
            return None  # 只有标题，无实质内容
        
        return "\n".join(lines)
```

### 3.2 Prompt 变化

**当前**（`translate_image` 中的 messages 构建）：

```python
messages = [
    {"role": "system", "content": system_prompt},
    {"role": "user", "content": [
        {"type": "image_url",
         "image_url": {"url": f"data:image/jpeg;base64,{base64_image}"}}
    ]}
]
```

**修改后**：

```python
# 构建系统提示词（可附加上下文）
system_content = system_prompt
if context:
    system_content = system_prompt + "\n\n" + context

messages = [
    {"role": "system", "content": system_content},
    {"role": "user", "content": [
        {"type": "image_url",
         "image_url": {"url": f"data:image/jpeg;base64,{base64_image}"}}
    ]}
]
```

> **为何不单独放一条 user message**：多模态模型中 system prompt 是最高优先级的指令性内容，把上下文放在 system 层确保 LLM 视其为角色设定的一部分。

### 3.3 上下文传递链路

```
BatchTranslationManager.start()
  ├─ context_engine.build_context(folder, chunk_start, window_size=5)
  ├─ context = "【前情提要】\n第5页：...\n第6页：..."
  └─ _process_chunk(paths, context)
       └─ TranslationWorker(path, context=context)
            └─ llm_engine.translate_image(path, context=context)
                 └─ system_prompt + context

手动翻译（F5）：
  window → controller.translate_page(path, context=None) → 无上下文，行为不变
```

**手动翻译不走上下文**：F5 翻译当前页时不传 context（`None`），保持原有独立翻译行为。只有批量翻译中才启用上下文——因为只有批量翻译中用户才明确表达了"我要连续翻译多页"的意图。

> 未来可扩展：在设置中添加"手动翻译也启用上下文"选项。

## 6. 风险点

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| context 过长导致 prompt 超 token 限制 | 低 | API 调用失败 | `max_chars=800` 硬限制（含中文字符约 400 字 plot），远低于常见模型 8K+ token 限制 |
| 前页翻译有误导致错误级联传播 | 中 | 后续页翻译质量下降 | v3.1 先做基础传递；v3.2 的回溯修正会审查和修复 |
| plot 字段缺失（旧版缓存） | 低 | 上下文不完整 | `build_context()` 跳过 plot 为空的页，不打乱上下文 |
| context 中包含格式标记干扰 LLM | 低 | 翻译输出异常 | plot 是自然语言总结，不包含 XML/Markdown 格式标记 |

## 7. 与 PDR-02 的集成点

`BatchTranslationManager` 在初始化后调用 `batch_manager.set_context_engine(context_engine)` 即可。集成代码如下（在 `window.py` 的 `_show_batch_dialog()` 中）：

```python
# 创建 context_engine（如果启用上下文）
if enable_context:
    context_engine = ContextEngine(translation_cache._db)  # 通过 cache 访问 database
    self.batch_manager.set_context_engine(context_engine)

self.batch_manager.start(target_paths, start, end)
```

`batch_manager._process_chunk()` 中每个 chunk 开始前：

```python
context = None
if self._context_engine:
    folder = os.path.dirname(chunk_paths[0])
    context = self._context_engine.build_context(folder, chunk_start_page)
```
