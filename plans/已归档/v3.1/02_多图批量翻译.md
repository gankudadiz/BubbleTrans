## 1. 目标

实现多页批量翻译：用户选择页码范围 → 分块并发翻译 → 实时进度反馈 → 可暂停/取消。翻译完成后结果自动写入缓存，翻阅即可查看。

## 2. 当前状态

### 2.1 现有翻译架构（v3.0 产物）

```
用户 F5 → window.translate_current_page()
          → controller.translate_page(path)
            → 创建 TranslationWorker → 启动 → 等待
            → _on_finished() → 写缓存 → 发信号 → window 更新 UI
```

`TranslationController` 一次只能管理一个 Worker，单页串行翻译。

### 2.2 现有预翻译（prefetch.py）

`PrefetchManager` 可以并发翻译后续 N 页，但它：
- 只预翻译"后续"页面（单向），不支持范围选择
- 静默执行，无进度条、无暂停/取消
- 直接从 window.py 获取文件列表，不暴露范围接口

### 2.3 改动范围

| 文件 | 改动类型 | 说明 |
|------|----------|------|
| `ui/batch.py` | **新建** | 批量翻译管理器（分块、调度、进度、暂停/取消） |
| `ui/window.py` | 修改 | 新增工具栏按钮/菜单 → 范围选择对话框 → 进度 UI |
| `ui/prefetch.py` | 可能复用 | 提取 `_run_worker` 并发逻辑供 batch.py 复用 |

### 3.2 分块调度逻辑

```
输入: file_paths = [p5, p6, ..., p25], start_page=5, end_page=25
    # 调用方（_show_batch_dialog）已从完整列表切片，start() 接收即用
↓
分块: chunks = [[p5-p9], [p10-p14], [p15-p19], [p20-p24], [p25]]
     # 每块记录 chunk_start_page（start_page + 块内偏移）
↓
遍历每个 chunk (chunk_start_page=5, 10, 15, 20, 25):
  ├─ 构建上下文（调用 context_engine，PDR-03 提供）
  ├─ 组内并发翻译（最多 concurrent_count 个 Worker 同时运行）
  ├─ 每页 page_index = chunk_start_page + 偏移量
  ├─ 等待整组完成
  ├─ 发射 progress_updated
  ├─ 检查暂停/取消标志
  └─ 继续下一 chunk
↓
全部完成 → batch_finished(success, failed)
```

**关键约束**：

| 参数 | 值 | 来源 |
|------|-----|------|
| chunk_size | 5 页 | 硬编码 |
| 并发数 | 2（默认复用 settings 中的预翻译并发数） | `config.py` |
| 单次最大范围 | 50 页 | 可配置 |
| 上下文窗口 | 前 5 页 plot（由 context_engine 提供） | PDR-03 |

**组内并发实现**：

```python
def _process_chunk(self, chunk_paths: list[str], context: str, chunk_start_page: int):
    """组内并发翻译，维护计数器 + 互斥锁，最后一个完成的 worker 才解除阻塞"""
    workers = []
    results = {}
    lock = threading.Lock()
    completed_count = [0]  # 用列表包装实现闭包内修改
    total = len(chunk_paths)

    def on_page_done(path, origin, translated, summary):
        with lock:
            results[path] = (origin, translated, summary)
            completed_count[0] += 1
            if completed_count[0] == total:
                event.set()  # 最后一个完成的 worker 才 set event

    def on_page_error(path, error_msg):
        with lock:
            results[path] = None  # 标记失败
            completed_count[0] += 1
            if completed_count[0] == total:
                event.set()

    for i, path in enumerate(chunk_paths):
        worker = TranslationWorker(path, context=context)
        page_index = chunk_start_page + i
        worker.finished.connect(
            lambda o, t, s, p=path: on_page_done(p, o, t, s)
        )
        worker.error.connect(
            lambda e, p=path: on_page_error(p, e)
        )
        workers.append(worker)

    for w in workers:
        w.start()

    # 等待所有 worker 完成（带超时防止死锁）
    event.wait(timeout=300)  # 5 分钟超时

    return results
```

> **设计决策**：不使用 `QThreadPool`，因为 `TranslationWorker` 继承自 `QThread`（不是 `QRunnable`），且 `QThreadPool` 无法精确控制并发数。改为手动管理 Worker 列表，维护运行中计数。

### 3.3 暂停/取消机制

```
pause(): 设置 _paused 标志
  → 当前运行中的 chunk 继续完成（不中断进行中的 Worker）
  → 下一个 chunk 开始前检查标志，阻塞等待 resume()

resume(): 清除 _paused 标志
  → 继续执行下一个 chunk

cancel(): 设置 _cancelled 标志
  → 终止所有运行中的 Worker（terminate + wait）
  → 不写入未完成的缓存
  → 已完成的 chunk 结果保留在缓存中
```

**类初始化参数**：

```python
def __init__(self, translation_cache, parent=None):
    super().__init__(parent)
    self._cache = translation_cache
    self._workers = []
    self._paused = False
    self._cancelled = False
    self._completed = 0
    self._failed = 0
    self._total = 0
    self._context_engine = None

def set_context_engine(self, engine):
    """设置上下文引擎（PDR-03 提供）"""
    self._context_engine = engine

def _write_cache(self, image_path, page_index, origin, translated, summary):
    """单页缓存写入，page_index 由调用方（chunk 遍历时从 start_page + 偏移量计算）"""
    try:
        mtime = os.path.getmtime(image_path)
        self._cache.set(image_path, mtime, {
            "original": origin,
            "translated": translated,
            "summary": summary,
        }, page_index=page_index)
        # save() 在 SQLite 方案下为 no-op，无需调用
    except Exception as e:
        _logger.warning(f"批量翻译缓存写入失败 ({image_path}): {e}")
```

```python
def _show_batch_dialog(self):
    """弹出批量翻译范围选择对话框"""
    total = len(self.file_browser.get_file_paths())
    if total == 0:
        self.status_bar.showMessage("请先打开图片文件夹或压缩包", 3000)
        return
    
    dlg = BatchRangeDialog(total, parent=self)
    if dlg.exec() != QDialog.DialogCode.Accepted:
        return
    
    start, end, enable_context = dlg.get_range()
    if start > end:
        start, end = end, start
    
    page_count = end - start + 1
    if page_count > 50:  # 上限检查
        QMessageBox.warning(self, "范围过大", "单次最多翻译 50 页")
        return
    
    # 启动批量翻译
    target_paths = self.file_browser.get_file_paths()[start:end+1]

    # 构建上下文引擎（如果启用）
    if enable_context:
        from engine.context import ContextEngine
        context_engine = ContextEngine(self.translation_cache._db)
        self.batch_manager.set_context_engine(context_engine)

    self.batch_manager.start(target_paths, start, end)
    self._show_batch_progress()
```

**操作 D**：新增进度回调：

```python
def _on_batch_progress(self, completed, total, current_page):
    self.batch_progress_bar.setValue(completed)
    self.batch_progress_bar.setMaximum(total)
    self.batch_progress_label.setText(f"{completed} / {total} 页已完成")

def _on_batch_finished(self, success, failed):
    self.batch_progress_frame.hide()
    msg = f"批量翻译完成: {success} 成功"
    if failed:
        msg += f", {failed} 失败"
    self.status_bar.showMessage(msg, 5000)
```

## 6. 风险点

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| Worker 池管理导致线程泄漏 | 中 | 内存增长、QThread 警告 | `_process_chunk` 完成后统一 `deleteLater`；`cancel()` 时 terminate 所有残留 Worker |
| 批量翻译中关闭窗口 | 中 | 崩溃或线程残留 | `window.closeEvent` 中调用 `batch_manager.cancel()` |
| API 限流导致批量失败率高 | 中 | 大量错误页 | 分块间加 1 秒冷却延迟；失败页跳过并记录，结束后列出 |
| 上下文为空时无差别执行 | 低 | 翻译质量下降 | context_engine 为 None 时仍可正常翻译（只是无上下文） |
| 暂停后长时间不恢复导致 Worker 超时 | 低 | Worker 挂着但不消耗资源 | Worker 已完成（isRunning=False），暂停只影响 chunk 调度 |

## 7. 不做的事

- **断点续传**：关闭重开后恢复上次中断点（v3.2）
- **失败页自动重试**：结束后列出失败清单，手动重试（当前方案）
- **翻译完成弹 Toast**：只更新状态栏，不打断用户（简洁优先）
