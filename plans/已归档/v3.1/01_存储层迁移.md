## 1. 目标

将翻译缓存从单文件 JSON（`translation_cache.json`）迁移到 SQLite（`utils/database.py`），同时保持 `TranslationCache` 对外的公共接口完全不变。

## 2. 当前状态

### 2.1 现有存储方案

- **文件**：`src/utils/cache.py`（267 行）
- **存储**：JSON 文件 → 内存 `OrderedDict`，启动时全量加载，每次修改后全量序列化落盘
- **LRU**：`MAX_ENTRIES = 500`，`OrderedDict.popitem(last=False)` 淘汰
- **线程安全**：`threading.RLock` 保护读写
- **接口**：`get()`, `set()`, `get_fallback()`, `clear_image()`, `clear()`, `save()`, `__len__()`
- **键格式**：`MD5(image_path + mtime)`

### 2.2 痛点

| 问题 | 影响 |
|------|------|
| 跨页查询需全表遍历 | v3.1 上下文引擎需要"取第 N-5 ~ N-1 页的 plot"，JSON 只能 O(n) 遍历 |
| 多并发写入有覆盖风险 | `translation_controller._on_finished()` 写缓存 + `prefetch.py` 预翻译写缓存，虽加了 RLock 仍有时序依赖 |
| 全量加载到内存 | 单文件膨胀时启动变慢 |

### 2.3 `cache.py` 调用方一览（需保持兼容）

| 调用位置 | 调用方式 | 说明 |
|----------|----------|------|
| `engine/llm.py:486` | `self.cache.get(image_path, mtime)` | 翻译前查缓存 |
| `engine/llm.py:557` | `self.cache.get_fallback(image_path)` | AI 返回空/异常时降级 |
| `engine/translation_controller.py:85` | `self._cache.set(current_path, mtime, {...})` | 翻译完成写缓存 |
| `engine/translation_controller.py:90` | `self._cache.save()` | 落盘 |
| `engine/translation_controller.py:157` | `self._cache.clear_image(image_path)` | 清除单页缓存 |
| `engine/translation_controller.py:161` | `self._cache.clear()` | 清除全部 |
| `engine/translation_controller.py:165` | `len(self._cache)` | 查询条数 |
| `ui/window.py` | 同 translation_controller 路径 | 间接调用 |

**结论**：只需重写 `cache.py` 内部实现，公共接口签名不变，所有调用方无需改动。

## 3. 设计方案

### 3.1 新建：`src/utils/database.py`

纯 Python 标准库 `sqlite3`，零额外依赖。

```python
# database.py 核心接口
class Database:
    def __init__(self, db_path=None): ...
    
    # 迁移
    def migrate_from_json(self, json_path: str) -> int: ...
    
    # CRUD（对 cache.py 暴露）
    def get(self, cache_key: str) -> dict | None: ...
    def set(self, cache_key: str, folder_path: str, page_index: int,
            image_path: str, original: str, translated: str,
            plot: str, notes: str) -> None: ...
    def get_fallback(self, image_path: str) -> dict | None: ...
    def clear_image(self, image_path: str) -> int: ...
    def clear_all(self) -> int: ...
    def count(self) -> int: ...
    
    # 上下文查询（对 context.py 暴露）
    def get_page_range(self, folder_path: str, 
                       start: int, end: int) -> list[dict]: ...
```

**Schema**：

```sql
CREATE TABLE IF NOT EXISTS translations (
    cache_key   TEXT PRIMARY KEY,       -- MD5(image_path + mtime)
    folder_path TEXT NOT NULL,          -- 所属文件夹
    page_index  INTEGER,               -- 文件夹内序号（0-based）
    image_path  TEXT NOT NULL,         -- 图片绝对路径
    original    TEXT,                  -- 原文
    translated  TEXT,                  -- 译文
    plot        TEXT,                  -- 剧情总结
    notes       TEXT,                  -- 翻译备注
    updated_at  TEXT NOT NULL          -- ISO 时间戳
);
CREATE INDEX IF NOT EXISTS idx_folder_page ON translations(folder_path, page_index);
CREATE INDEX IF NOT EXISTS idx_image_path ON translations(image_path);
```

**连接管理**：
- WAL 模式：`PRAGMA journal_mode=WAL;`
- 写入重试：最多 3 次，间隔 100ms
- 连接生命周期：`__init__` 创建 → `close()` 销毁，不支持连接池（单进程应用无此需求）

### 3.2 LRU 淘汰策略

当前 `MAX_ENTRIES = 500`。SQLite 方案改为：

- 默认上限 2000 条（可配置）
- 每次 `set()` 后检查 `COUNT(*)`，超出时按 `updated_at ASC` 删除最旧记录
- 删除数量 = 超出数量 + 10%（一次性批量清理，避免每次 set 都删）

### 3.3 修改：`src/utils/cache.py`

**核心原则**：公共接口签名不变，只改内部实现。

改动对照：

| 方法 | 当前实现 | 新实现 |
|------|---------|--------|
| `__init__` | `json.load()` 全量加载到 `OrderedDict` | 初始化 `Database` 实例，检测 JSON → 执行迁移 |
| `_load()` | 读 JSON 文件 | 删掉（或改为 `pass`） |
| `save()` | `json.dump()` 全量序列化 | **no-op**（SQLite 每次 set 即时落盘） |
| `get(image_path, mtime)` | `_data[key]` + LRU move_to_end | `self._db.get(key)` 查询 |
| `set(image_path, mtime, data)` | `_data[key] = entry` + LRU 淘汰 | `self._db.set(...)` 写入 + LRU 淘汰 |
| `get_fallback(image_path)` | 遍历 `_data` 找匹配 image_path | `self._db.get_fallback(image_path)` |
| `clear_image(image_path)` | 遍历 `_data` 删匹配项 | `self._db.clear_image(image_path)` |
| `clear()` | `_data.clear()` → `save()` | `self._db.clear_all()` |
| `__len__()` | `len(self._data)` | `self._db.count()` |
| `_make_key()` | 保留 | 保留 |
| `_is_error_entry()` | 保留 | 保留 |

**关键：`folder_path` 和 `page_index` 的计算**：

当 `set()` 被调用时，`folder_path` 通过 `os.path.dirname(image_path)` 内部计算。

**`page_index` 的获取——必须在翻译启动时捕获，不能在回调中查询**：

`_on_finished()` 是 Worker 线程完成后的异步回调。翻译耗时若干秒，在此期间用户可能已经翻到其他页面。如果在回调中查询 `FileBrowser.currentRow()`，拿到的将是用户当前所在的页码，而非发起翻译时的页码——导致翻译结果被标记为错误的 `page_index`。

**正确做法**：在 `translate_page()` 调用时立即捕获 `page_index`，与 `_translating_image_path` 一并存储为 `self._translating_page_index`。`_on_finished()` 中直接使用该值。无需任何新方法，只需利用已有的 `FileBrowser.current_file_index` 属性（`file_browser.py:142-144`）。

### 3.4 迁移流程

```
启动 → TranslationCache.__init__()
  ├─ 检查 translation_cache.json 是否存在
  │   ├─ 存在 → database.migrate_from_json(json_path)
  │   │         ├─ 逐条读取 JSON
  │   │         ├─ 插入 SQLite（用 INSERT OR REPLACE 避免重复）
  │   │         ├─ 比对条数：JSON 条数 vs SQLite 条数
  │   │         ├─ 一致 → 重命名 JSON 为 .bak
  │   │         └─ 不一致 → 报错，保留 JSON 不动
  │   └─ 不存在 → 跳过
  └─ 之后所有读写走 SQLite
```

**迁移时的 `folder_path` / `page_index` 回填**：

JSON 旧数据中每个 entry 有 `image_path` 字段，迁移时可：
- `folder_path = os.path.dirname(image_path)`
- `page_index`：因 JSON 没有此字段，迁移时设为 `NULL`。后续首次翻译该页时自动补全。

**操作 B**：`__init__` 改为：

```python
def __init__(self, cache_path=None):
    # 路径计算逻辑不变
    if cache_path is None:
        if getattr(sys, 'frozen', False):
            _project_root = os.path.dirname(os.path.abspath(sys.executable))
        else:
            _dir = os.path.dirname(os.path.abspath(__file__))
            _project_root = os.path.dirname(os.path.dirname(_dir))
        cache_path = os.path.join(_project_root, CACHE_FILENAME)
    
    self.cache_path = cache_path
    self._lock = threading.RLock()
    
    from utils.database import Database
    db_path = cache_path.replace('.json', '.db')
    self._db = Database(db_path)
    
    # 检测并执行 JSON → SQLite 迁移
    json_path = cache_path  # translation_cache.json
    if os.path.exists(json_path):
```

**操作 C**：逐个方法委托给 `self._db`：

```python
def get(self, image_path, mtime):
    with self._lock:
        key = self._make_key(image_path, mtime)
        return self._db.get(key)

def set(self, image_path, mtime, data, page_index=None):
    with self._lock:
        import time
        key = self._make_key(image_path, mtime)
        folder_path = os.path.dirname(image_path)
        self._db.set(
            cache_key=key,
            folder_path=folder_path,
            page_index=page_index,  # None 时 SQLite 存 NULL
            image_path=image_path,
            original=data.get("original", ""),
            translated=data.get("translated", ""),
            plot=data.get("summary", {}).get("plot", ""),
            notes=data.get("summary", {}).get("notes", ""),
        )

def save(self):
    pass  # SQLite 即时落盘，无需显式保存

def get_fallback(self, image_path):
    with self._lock:
        return self._db.get_fallback(image_path)

def clear_image(self, image_path):
    with self._lock:
        return self._db.clear_image(image_path)

def clear(self):
    with self._lock:
        return self._db.clear_all()

def __len__(self):
    with self._lock:
        return self._db.count()
```

**操作 D**：保留 `_make_key()` 和 `_is_error_entry()` 方法不变。

## 6. 风险点

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| JSON 迁移数据不完整 | 低 | 部分翻译缓存丢失 | 迁移后比对条数；不一致时弹出错误对话框告知用户、保留 JSON 不动、程序正常运行（使用空缓存，旧 JSON 需手动修复后删除 .db 重试） |
| `set()` 签名变化导致调用方遗漏 | 中 | page_index 为 NULL | `page_index` 参数默认 `None`，向后兼容；lint 检查所有调用方 |
| SQLite 写入并发冲突 | 低 | 数据丢失 | WAL 模式 + 写入重试（最多 3 次） |
| 旧 JSON 残留导致每次启动都尝试迁移 | 低 | 启动变慢 | 迁移成功后立即重命名为 `.bak`，检查时跳过 `.bak` 文件 |
| `save()` 变 no-op 导致调用方期望落盘 | 低 | 调用方误认为数据未落盘 | `save()` 保留为空方法，不做任何操作（不再抛异常） |

## 7. 回滚方案

若出现不可修复的 SQLite 损坏：
1. 删除 `translation_cache.db`
2. 还原 `translation_cache.json.bak` → `translation_cache.json`
3. 回退 `cache.py` 到旧实现
4. 重启程序
