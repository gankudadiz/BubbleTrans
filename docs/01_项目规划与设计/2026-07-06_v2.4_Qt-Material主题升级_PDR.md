# BubbleTrans v2.4 Qt-Material 主题升级 PDR

**日期**：2026-07-06
**版本**：v2.4
**状态**：设计中

---

## 1. 升级动机

v2.3 功能已经完整可用，但 UI 仍停留在「原生 Qt 控件 + 内联 QSS」阶段，视觉粗糙程度与功能完整度不匹配：

- **主题不统一**：主窗口是深色主题，设置弹窗却是系统浅色主题，切换时割裂感严重
- **控件原生**：QPushButton / QComboBox / QLineEdit / QScrollBar 全部使用 Qt 默认渲染，无圆角、无阴影、无 hover 过渡
- **样式散落**：约 17 处 `setStyleSheet("""...""")` 散布在 3 个 `.py` 文件中，无全局主题文件、无颜色变量体系
- **间距随意**：模块间距、内边距无统一栅格，右侧翻译面板各区块视觉权重混乱
- **选中反馈弱**：文件列表高亮仅靠文字颜色区分，不够显眼

目标：**用最小代码改动量，统一深色主题 + 提升控件精致度**。

## 2. 现状分析

### 2.1 UI 架构现状

| 维度 | 现状 |
|------|------|
| GUI 框架 | PyQt6（纯原生控件） |
| 样式方式 | 内联 QSS 字符串，散落于 `src/ui/window.py`、`src/ui/canvas.py`、`src/ui/settings.py` |

| 动画 | 少量自写动画：骨架屏 `QPropertyAnimation`、按钮 `QTimer` 动画 |

| 优先级 | 问题 | 位置 |

| **P1** | 右侧翻译面板模块间距不一致 | `window.py` 右侧 QWidget |

## 3. 方案选型

### 3.1 候选方案对比

| 方案 | 侵入性 | 视觉效果 | 依赖数 | 许可证 | 维护状态 |
|------|--------|----------|--------|--------|----------|
| **Qt-Material** | ⭐ 极低（1行代码） | Material Design，圆角+阴影 | 1个（Jinja2） | BSD-2 | 活跃 |
| PyQt-Fluent-Widgets | ⭐⭐⭐ 高（需换控件类） | Fluent Design，最佳 | 3~6个 | GPL-3.0 | 非常活跃 |
| QDarkStyleSheet | ⭐ 极低 | 朴素深色主题 | 1个+qtpy | MIT | 维护模式 |
| 集中化 QSS 自建 | ⭐⭐ 中 | 取决于投入 | 0 | — | — |

### 3.2 选型决策：Qt-Material

**核心理由**：

1. **极低侵入**：只需在 `main.py` 的 `QApplication` 创建后加一行 `apply_stylesheet(app, theme='dark_teal.xml')`，不需要替换任何控件类
2. **统一覆盖**：`apply_stylesheet` 作用于整个 QApplication，主窗口、设置弹窗、QMessageBox、右键菜单全部统一主题
3. **Material Design**：自带圆角、阴影、规范间距、hover/pressed 状态过渡，视觉提升立竿见影
4. **20+ 主题可选**：`dark_teal` / `dark_blue` / `dark_purple` / `dark_amber` 等，一行切换
5. **许可证宽松**：BSD-2，无商业限制
6. **依赖轻量**：仅 `Jinja2`（15KB），`pip install qt-material` 一条命令

**为什么不选其他**：

- PyQt-Fluent-Widgets 效果最好但 GPL-3.0 有传染性风险，且需要把 `QMainWindow` 改为 `FluentWindow`、`QPushButton` 改为 `PrimaryPushButton` 等，侵入性太高
- QDarkStyleSheet Qt6 支持标称「可能不稳定」，且只是变暗，不加圆角阴影
- 集中化 QSS 自建费时且效果不确定

### 3.3 不做的事

- 不引入 QtAwesome 图标库（v2.4 仅做主题统一，图标可后续迭代）
- 不重写控件类（保留现有 `QMainWindow` / `QDialog` / `QMessageBox` 全部不变）
- 不删除现有内联 QSS（Qt-Material 优先级高于内联样式，不冲突；待验证后再决定是否逐步清理）

## 4. 实施方案

### 4.1 核心改动

**`src/main.py`**（改动量：2 行）：

```python
from qt_material import apply_stylesheet

app = QApplication(sys.argv)
# 应用 Material Design 深色主题，统一所有窗口
apply_stylesheet(app, theme='dark_teal.xml')
```

### 4.2 可选微调

```python
extra = {
    'font_family': 'Microsoft YaHei',   # 中文字体
    'density_scale': '-1',              # 稍微紧凑，适配信息密集型界面
}
apply_stylesheet(app, theme='dark_teal.xml', extra=extra)
```

### 4.3 依赖变更

`requirements.txt` 增加一行：

```
qt-material>=2.14
```

### 4.4 影响范围

| 文件 | 改动类型 | 说明 |
|------|----------|------|
| `src/main.py` | +2 行导入和调用 | 应用全局主题 |
| `requirements.txt` | +1 行依赖 | 新增 qt-material |
| `src/ui/window.py` | 无需改动 | 主题自动覆盖 |
| `src/ui/settings.py` | 无需改动 | 主题自动覆盖 |
| `src/ui/canvas.py` | 无需改动 | 主题自动覆盖 |

### 4.5 实施步骤

1. `pip install qt-material`
2. 在 `src/main.py` 加入 2 行
3. 启动验证主窗口、设置窗口、弹窗主题是否统一
4. 切换 3~5 个主题配色，选定最终方案
5. 检查内联 QSS 是否有与 Qt-Material 冲突的部分，如有冲突则移除对应 `setStyleSheet` 调用
6. 更新 `README.md` 版本号为 v2.4

## 6. 风险与限制

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 内联 QSS 与 Qt-Material 冲突 | 中 | 个别控件样式异常 | 逐个检查 17 处内联样式，冲突的删除 |
| 骨架屏/自定义动画异常 | 低 | 动画颜色不匹配 | Qt-Material 只改 QSS，不影响 QPropertyAnimation 逻辑 |
| 部分控件未覆盖 | 低 | 个别元素仍是原生样式 | 通过 `extra` 自定义 CSS 或保留必要内联 QSS |
| density_scale 影响阅读体验 | 低 | 信息过密或过疏 | 对比 -1/0/+1 三档后选定 |
