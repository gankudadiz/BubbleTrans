# BubbleTrans v2.2 ~ v2.3 体验优化规划

**创建日期**：2026-06-30
**当前状态**：规划中
**目标版本**：v2.2（体验修复）+ v2.3（阅读体验）

---

## 背景

v2.0 完成了纯多模态架构重构，v2.1 完成了右侧面板优化与 AI 总结功能。核心翻译流程已跑通，当前重点是从"能用"提升到"好用"，补齐阅读漫画场景下的基础体验短板。

---

## v2.2 — 体验修复

### F1. 翻译缓存

**问题**：每次重启程序，之前翻译过的页面全部丢失，重新翻译既耗时又费 API 额度。

**方案**：
- 按图片文件路径 + 修改时间 hash 作为缓存键
- 缓存内容：原文、译文、剧情总结、翻译备注
- 程序启动时加载缓存索引，翻译前先查缓存
- 缓存上限 500 条，LRU 淘汰
- 缓存存储位置：`config.json` 同级目录下的 `translation_cache.json`

### F2. 压缩包格式支持

**问题**：漫画圈最主流的格式是 `.cbz` / `.cbr`（即 zip / rar 压缩包），当前只能打开松散图片文件夹。

**方案**：
- 优先支持 `.cbz`（zip 格式），技术成本低
- 支持 `.cbr`（rar 格式），需评估是否引入 `unrar` 依赖
- 打开压缩包时解压到临时目录，关闭时清理
- 文件列表展示压缩包内图片，翻页逻辑不变
- `.zip` 格式一并支持（部分用户用 zip 打包漫画）

### F3. 翻译进度细化

**问题**：当前状态栏只显示"翻译中…"，用户不知道还要等多久、在哪个阶段。

**方案**：
- 状态栏分阶段提示：「正在编码图片…」→「等待 API 响应…」→「正在解析结果…」→「生成总结…」
- 翻译按钮增加加载动画（旋转图标）
- 右侧面板显示骨架屏占位，避免"白屏等待"感

## v2.3 — 阅读体验

### F4. 快捷键提示

**问题**：用户不知道有哪些快捷键，当前完全无引导。

**方案**：
- 首次启动时弹出快捷键引导浮层（可关闭，不再提示）
- 状态栏常驻提示核心快捷键：← → 翻页 | 右键框选 | 滚轮缩放
- Settings 按钮 hover 时 tooltip 也补充说明

### F5. 文件列表搜索

**问题**：文件夹内有几百张图时，滚动找图效率极低。

**方案**：
- 文件列表顶部增加搜索框
- 实时过滤文件名（模糊匹配，不区分大小写）
- 搜索时高亮匹配文本
- 清空搜索框恢复完整列表

### F6. 最近打开文件夹

**问题**：每次启动都要重新选择文件夹，无法快速回到上次阅读的漫画。

**方案**：
- 记住最近 10 个打开过的文件夹路径
- 工具栏 Open Folder 旁增加下拉箭头，列出最近文件夹
- 启动时自动打开上次阅读的文件夹（可选，默认开启）
- 已不存在的路径自动从列表中移除

### F7. 阅读位置记忆

**问题**：每次重启程序，不仅要重新选文件夹，还要手动翻到上次阅读的页码，打断阅读心流。

**方案**：
- 记住每个文件夹最后一次阅读的页码（文件列表中的索引）
- 打开文件夹时自动跳转到上次阅读的页码
- 切换文件夹时实时更新当前文件夹的阅读位置
- 阅读位置与最近打开文件夹（F6）共享同一份持久化数据
- 存储位置：`config.json` 中新增 `last_positions` 字段，结构为 `{ "文件夹路径": 页码索引 }`
- 上限 20 条，超出淘汰最久未访问的记录

---

## 实施顺序建议

```
v2.2（先做，解决"痛点"）
  F1 翻译缓存 → F2 压缩包支持 → F3 进度细化

v2.3（后做，做"爽点"）
  F4 快捷键提示 → F5 文件列表搜索 → F6 最近打开文件夹 → F7 阅读位置记忆
```

每完成一个 F 即可单独提交，不必等全部做完。

---

## 不在本期范围

以下功能有价值，但优先级不高，留待后续版本：
- 翻译结果导出（TXT/MD/图片叠加）
- 双页阅读模式
- 翻译结果可编辑
- 深色/浅色主题切换
