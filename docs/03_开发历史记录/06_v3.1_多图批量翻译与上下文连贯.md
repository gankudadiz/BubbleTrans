# 06 v3.1 多图批量翻译与上下文连贯

**日期**：2026-07-10
**提交**：`865a71b` feat: v3.1 多图批量翻译与上下文连贯

## 需求背景

单页独立翻译导致跨页角色名漂移和剧情断层；手动翻译几十页漫画繁琐重复；翻译缓存仅存于内存 JSON，跨会话也无法用于压缩包场景。

## 实现要点

1. **SQLite 存储层**（`database.py` 338 行）：替代内存 JSON，支持跨页范围查询（`get_page_range`）、WAL 模式、LRU 淘汰、旧 JSON 自动迁移。
2. **固定解压目录**：`archive.py` 改用 MD5(压缩包路径) 生成确定性临时目录，配合 `_make_key` 中 mtime 取整，实现压缩包翻译缓存的跨会话持久化。
3. **批量翻译管理器**（`batch.py` 238 行）：分块调度（每 5 页一组）、组内并发、暂停/取消、进度条。靠 20ms 高频 `processEvents` 保持 UI 流畅。
4. **滑动窗口上下文**（`context.py` 42 行）：翻译第 N 页时将前 5 页 plot 注入 system prompt，解决角色名漂移和剧情断层。
5. **耗时提示**：>15 页时弹出区间预估（乐观/悲观系数），让用户做知情选择。
6. **时序修复**：`page_index` 在 `translate_page()` 调用时立即捕获，避免异步回调中查询到错误页码。

## 执行修正（冒烟测试阶段）

- 压缩包翻译缓存从不写入：`_is_temp_image` 标志替代路径推断。
- 批量翻译 UI 卡顿：poll 间隔 500ms→20ms，进度条前置显示。
- 缓存退出丢失：`Database.close()` 加 WAL checkpoint。
- 跨会话 mtime 漂移：`_make_key` 中 `int(mtime)` 消除子秒级噪声。
