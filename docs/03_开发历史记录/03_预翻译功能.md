## 需求背景

翻译单页需等待 30~40 秒 LLM 响应。翻页看画容易剧透，阅读体验被割裂。解决思路：用户翻页前，后台提前翻译后续页面，翻到时直接命中缓存。

## 实现要点

### PrefetchManager（`src/ui/prefetch.py`）
- 后台并发队列，默认 2 并发，去重判断（跳过缓存命中 + 已在队列 + 正在翻译）
- `enqueue(paths)` 入队，`promote(path)` 用户 F5 时移出队列避免重复，`clear()` 清空
- 完成时写入 `TranslationCache`，发射 `progress_changed` / `page_completed` / `all_completed` 信号

### UI 集成（`src/ui/window.py`）
- 新增"预翻译 ▼"下拉按钮 → 下 1/3/5 页
- `Ctrl+Shift+F5` 快捷键 → 预翻译下 3 页
- F5 主翻译时调用 `promote()` 避免与预翻译竞争
- 预翻译完成页面时如果用户正好翻到该页，自动刷新右侧面板

### 配置（`src/ui/settings.py`）
- 设置页新增预翻译开关和并发数配置

## 验证收口

- 打开漫画文件夹 → 预翻译下 3 页 → 翻到第 2 页时翻译结果已就绪
- 末页时预翻译按钮自动禁用
- F5 + Ctrl+Shift+F5 交错操作无竞争问题
- 自动预翻译模式下翻页自动触发

## 本阶段收益

- 从"翻页=等待"升级为"翻页=即时阅读"，阅读流不被 LLM 延迟打断
- 缓存命中率提升：用户还没翻到的页面已被预翻译缓存
