## 需求背景

v2.3 功能完整但 UI 粗糙：原生 Qt 控件无圆角阴影、主窗口深色 vs 设置弹窗浅色割裂、内联 QSS 散落 17 处不可维护。需要以最小改动量统一视觉效果。

## 实现要点

### 主题升级
- 集成 `qt-material` 库，`src/main.py` 加一行 `apply_stylesheet()` 全局应用 Material Design 主题
- 通过 `app.styleHints().colorScheme()` 自动跟随系统明暗模式（dark_teal / light_teal）
- `requirements.txt` 新增 `qt-material>=2.14`，`setup_check.py` 补依赖检测保证开箱即用
- README 版本号 v2.1→v2.4，截图和依赖列表同步更新

### 交互优化
- **左侧侧边栏**：移除 `setFixedWidth(200)`，改 `setMinimumWidth(120)`，QSplitter 拖拽调整宽度
- **右侧工具栏**：拆为双行布局——主操作行（翻译+预翻译）+ 辅助行（清除缓存+语言标签），按钮不再截断
- **分段切换按钮**：缩小 padding/font-size 释放垂直空间
- **预翻译按钮**：修复手动箭头与 QToolButton 原生箭头重复问题；选完后显示"预翻译中…"执行态反馈
- **清除缓存**：QPushButton → QToolButton 下拉菜单，新增"清除当前页缓存"（调用已有的 `clear_image()`）
- **类型修复**：`llm.py` cache 属性加 `Optional[TranslationCache]` 标注

## 验证收口

- 主窗口 + 设置弹窗 + QMessageBox 统一 Material Design 深色主题
- 系统切亮色模式后自动切换 light_teal 主题
- 左侧侧边栏可拖拽调整宽度，最小 120px
- 工具栏按钮文字不再截断
- 清除当前页缓存 → 右侧面板立即清空，重新翻译走 API

## 本阶段收益

- UI 从"功能能跑"提升为"精致可看"，Material Design 圆角阴影统一风格
- 开箱即用：`run.bat` 自动检测安装 qt-material
- 所有交互优化在 `window.py` 单文件内完成，无架构改动
