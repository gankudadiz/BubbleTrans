# 01 右侧面板优化与当页AI总结

**日期**：2026-06-29
**关联提交**：`c45bb93`

---

## 需求背景

v2.0 右侧翻译面板将原文和译文上下等分排列，原文区域对大多数使用场景是冗余信息。同时，用户希望利用多模态 LLM 的视觉理解能力，在翻译之外获取当页剧情脉络和翻译决策说明。

---

## 实现要点

### UI 改造（`src/ui/window.py`）

- 移除独立原文/译文 QTextEdit，替换为分段切换按钮（Segmented Control）+ 共享文本区
- 原文和译文通过 `[原文│▓译文▓]` 按钮切换显示，默认选中译文
- 新增当页总结区（剧情摘要 + 翻译备注），通过 QSplitter 与文本区分割，用户可拖动调整比例
- TranslationWorker 信号从 `pyqtSignal(str, str)` 扩展为 `pyqtSignal(str, str, dict)`

### LLM 引擎改造（`src/engine/llm.py`）

- 系统提示词追加 `<总结>/<剧情>/<备注>` 输出格式指令
- `_parse_response()` 新增总结提取逻辑
- `translate_image()` 返回值从二元组改为 `(原文, 译文, summary_dict)` 三元组
- 错误路径降级返回空 dict，UI 层显示"本页暂未生成总结"

---

## 改动范围

| 文件 | 改动 |
|------|------|
| `src/ui/window.py` | UI 布局重构 + 信号适配 |
| `src/engine/llm.py` | 提示词扩展 + 解析器扩展 + 返回类型变更 |

`canvas.py`、`settings.py`、`main.py` 不受影响。

---

## 验证收口

- 分段切换按钮视觉符合设计规范，点击切换流畅
- 翻译完成后自动切到译文 tab，总结区同步显示
- 框选翻译和整页翻译两种路径均正常工作
- QSplitter 拖动调整文本区/总结区高度比例
- 旧版 LLM 不返回总结时，UI 降级显示提示

---

## 本阶段收益

- 译文获取完整展示空间，不再被原文挤占
- 每页翻译附带剧情脉络和翻译决策说明，提升阅读体验
- 一次 API 调用完成三件事（原文 + 译文 + 总结），不增加额外耗时
